# KNOWN_RISKS

위험은 카테고리별로 정리. 각 항목에 영향/완화책을 둔다.

## 1. AI 정확도/환각
- 위험: Ollama gemma4:e4b가 OCR에 없는 정보를 만들거나 카테고리/금액을 잘못 추정.
- 영향: 잘못된 후보 생성. 다만 승인형 구조라 최종 저장은 사용자 결정.
- 완화: zod 검증, raw_text_basis substring 검증, 학습 규칙 후처리, confidence 표시, warnings 표기.

## 2. AI 서버 가용성
- 위험: 로컬 PC가 꺼져 있으면 분석 불가.
- 완화: 명확한 안내, OCR/학습 규칙 기반 fallback 후보, 수동 입력 경로.

## 3. Vercel 함수 시간/메모리 한계
- 위험: OCR/AI 호출이 길어지면 타임아웃.
- 완화: Node Runtime 사용, 비동기 분리(클라이언트 폴링), 큰 작업은 별도 Node 서버로 분리 검토.

## 4. Tesseract.js 한계
- 위험: 한글 영수증 인식률 한계, 흐릿한 사진/영수증에서 실패.
- 완화: 사용자 텍스트 수정 허용, 향후 PaddleOCR/Clova 등 대체 검토.

## 5. 민감정보 누출
- 위험: 카드/계좌/승인번호가 마스킹 누락으로 저장/로그/AI 요청에 포함.
- 완화: 단일 마스킹 모듈, 모든 외부 출력 통과 강제, 단위 테스트, 로그 정책.

## 6. RLS 누락
- 위험: 다른 사용자 데이터 노출.
- 완화: 모든 사용자 소유 테이블 RLS 강제, RLS 단위 테스트 케이스, 코드 리뷰 체크.

## 7. service role key 노출
- 위험: 클라이언트 번들에 포함 시 모든 RLS 우회 가능.
- 완화: 서버 전용. 변수명 prefix가 NEXT_PUBLIC_가 아니어야 한다는 원칙. 빌드 산출물 grep 점검.

## 8. Storage 정책 오류
- 위험: 다른 사용자 폴더에 업로드/조회 가능.
- 완화: 폴더 명명 규칙(`{user_id}/...`) + storage 정책 단위 테스트.

## 9. 무료 플랜 한도
- 위험: Supabase 0.5GB DB / 1GB Storage, Vercel Hobby 함수/대역폭 한도.
- 완화: 원본 이미지 자동 정리 옵션, 압축, 사용량 모니터링, 알림.

## 10. AI 학습 데이터 민감성
- 위험: global_learning_rules에 PII 유입.
- 완화: 자동 갱신 금지(또는 익명화 파이프라인 통과만), 가맹점 정규화 규칙, K-익명성.

## 11. 중복 거래 자동 저장
- 위험: 자동 승인이 잘못된 중복을 만들면 통계 왜곡.
- 완화: 자동 승인 금지, 의심 항목은 일괄 승인 제외.

## 12. 시간대(TZ)
- 위험: KST/UTC 혼재로 월 경계가 어긋남.
- 완화: DB는 UTC 저장, 클라이언트 표시 KST. 집계는 KST 경계로 변환.

## 13. 모바일 카메라/파일 입력 호환성
- 위험: 일부 브라우저에서 capture 옵션 무시.
- 완화: fallback 파일 선택 input, 사용자 안내, 다중 업로드 지원.

## 14. 색상 대비/접근성
- 위험: 핑크 톤이 옅어 텍스트 가독성 저하.
- 완화: 본문 텍스트는 진한 회색, 핑크 위 텍스트는 흰색, AA 대비 점검.

## 15. 빌드/배포 실패
- 위험: 환경변수 오타로 production 깨짐.
- 완화: Vercel 배포 전 체크리스트, preview에서 e2e 1회.

## 16. AI 응답 비결정성
- 위험: temperature가 높으면 같은 입력에 다른 결과.
- 완화: temperature 0~0.2 사용, format=json, 검증기 강제.

## 17. 다중 사용자 운영
- 위험: AI 서버가 1인 PC 기반이면 가용성/개인정보/동시성 문제. 외부 사용자 트래픽이 들어오면 즉시 한계.
- 완화: 24/7 가동 가능한 전용 머신 또는 GPU 서버로 분리, Cloudflare Tunnel + 토큰 인증, 동시 요청 큐잉/Rate Limit, 사용자별 입력 격리, 사용량 모니터링.

## 18. 이미지 보관 장기화
- 위험: 영수증 이미지 누적으로 Storage 한도 초과 + 개인정보 보관 부담.
- 완화: 사용자별 자동 정리 옵션 + 명시적 삭제 + 마스킹된 텍스트 위주 보관.

## 19. 회귀(Regression)
- 위험: 새 기능 추가 시 기존 흐름이 깨짐.
- 완화: 단위/통합/E2E + AI 하네스. 핵심 플로우 시각 회귀.

## 20. 커뮤니케이션/문서 동기화
- 위험: 코드와 docs가 어긋남.
- 완화: PR 템플릿에 "관련 docs 갱신 여부" 체크. 큰 결정은 docs 우선 갱신.
