# RESPONSIVE_DESIGN_PLAN

## 브레이크포인트
| 라벨 | 폭 | Tailwind |
|---|---|---|
| Mobile | 360–767 | 기본 |
| Tablet | 768–1023 | `md` |
| Desktop | 1024–1279 | `lg` |
| Wide | 1280+ | `xl` |

## 화면별 반응형 레이아웃

### 대시보드
- Mobile: 1~2열 카드, 차트는 폭 100%.
- Tablet: 2열 카드, 차트 1열.
- Desktop: 3~4열 카드 + 좌차트/우목록.
- Wide: 4열 카드 + 차트 더 넉넉히.

### 거래내역
- Mobile: 카드 리스트(상단 검색/필터는 접기/펼치기 패널 또는 바텀시트).
- Tablet: 카드 리스트(2열 그리드 옵션) 또는 표(컬럼 줄임).
- Desktop/Wide: 표(전체 컬럼) + 좌측 사이드바.

### AI 업로드
- Mobile: 상단 큰 업로드 버튼 + 카메라 입력 + 진행 상태 풀 폭.
- Tablet 이상: 좌(업로드/카메라) + 우(OCR 미리보기 + 진행 상태). 하단 후보 표/카드.
- Desktop: 1280+에서는 후보 영역을 표로.

### 분석 결과 후보
- Mobile: 카드 리스트, 일괄 버튼은 sticky bottom.
- Tablet: 2열 카드 그리드.
- Desktop: 표 + 상단 일괄 버튼.

### 카테고리/결제수단 관리
- Mobile: 목록 + 모달/바텀시트 편집.
- Desktop: 목록 + 우측 폼 분할.

### 설정
- Mobile: 섹션 접기/펼치기.
- Desktop: 섹션 카드 그리드.

## 컴포넌트 전환 패턴
- `TransactionTable` (md+) ↔ `TransactionCardList` (sm).
- `CandidateTable` (md+) ↔ `CandidateCardList` (sm).
- `Sidebar` (md+) + `BottomNav` (sm) — 둘 중 하나만 노출.
- 컴포넌트는 별도 구현하고 `useBreakpoint()` 또는 `<ResponsiveSwitch sm={...} md={...} />`로 스위치.

## 터치/입력 가이드
- 터치 영역 최소 44×44 px.
- input 높이 44 이상, button height 44 이상.
- 폰트: 모바일 본문 16px(자동 zoom 방지).

## 표/카드 변환 규칙
- 모바일에서 표는 가로 스크롤 대신 카드로 전환.
- 꼭 표가 필요한 화면은 가로 스크롤을 허용하되 sticky 첫 컬럼 적용.

## 이미지/미리보기
- OCR 미리보기 텍스트는 `max-h` + 접기/펼치기.
- 영수증 이미지 미리보기는 `object-contain`으로 비율 유지, 모바일에서는 풀폭.

## 모바일 업로드 UX
- `<input type="file" accept="image/*" capture="environment" multiple>` — 카메라 직접 촬영 + 다중 선택.
- 업로드 직전 썸네일 미리보기.
- OCR 진행 중 명확한 로딩 표시.
- 실패 시 다시 업로드/수동 입력 안내.

## 모바일 승인 UX
- 카드 하단 큰 버튼: 수정 / 제외 / 승인.
- 일괄 승인은 sticky bottom bar. 의심 항목은 자동 제외.
- 중복 의심·확인 필요 배지로 실수 승인 방지.
- 승인 후 토스트 피드백.

## 반응형 테스트 폭
- 360 / 390 / 768 / 1024 / 1280 / 1440 — 각 폭에서 위 화면들을 모두 확인.

## QA 체크리스트
- 로그인 깨짐 없음.
- 대시보드 카드 정상 배치.
- 거래내역 표↔카드 전환 정상.
- AI 업로드 모바일에서 사용 가능.
- OCR 미리보기 넘침 없음.
- 후보 카드/표 정상.
- 승인/수정/제외 터치 가능 크기.
- 카테고리/결제수단 관리 모바일 편의성.
- 설정 위험 버튼 실수 방지.
- 모바일 하단 네비/PC 사이드바 정상 동작.
- 화이트+핑크 톤 가독성·상태색 충돌 없음.
