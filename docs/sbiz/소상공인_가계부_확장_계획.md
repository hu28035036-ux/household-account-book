# 소상공인용 가계부 확장 계획서

작성일: 2026-05-11  
대상 프로젝트: AI 가계부  
목표: 기존 개인/가족 가계부를 "소상공인 간단 지출 정리 보조" 영역으로 확장한다.

---

## 1. 한 줄 방향

소상공인용 확장은 세무 신고 자동화가 아니라, 영수증 사진·카드내역·계좌내역·엑셀 파일을 모아 AI가 지출 후보를 만들고 사용자가 승인한 항목만 월별 비용 장부로 정리하는 기능으로 시작한다.

추천 포지션:

> 사장님이 영수증과 카드/계좌 내역을 올리면, AI가 지출 후보를 정리하고 사장님이 승인한 항목만 월별 비용표와 세무사 전달용 자료로 모아주는 서비스.

피해야 할 포지션:

- 세금 자동 신고
- 절세 자동 판단
- 경비 인정 여부 확정
- 회계 프로그램 완전 대체
- 세무 대리 서비스처럼 보이는 문구

---

## 2. 왜 이 확장이 맞는가

### 좋은 것

- 현재 앱에 이미 거래, 카테고리, 결제수단, OCR, AI 후보, 승인, CSV/XLSX 가져오기, 내보내기, 모임/household 구조가 있다.
- 소상공인 지출 정리는 기존 가계부 기능을 크게 버리지 않고 문구와 기본 카테고리, 리포트만 바꿔도 MVP를 만들 수 있다.
- 핵심 사용자 불편이 명확하다. 영수증이 흩어지고, 카드내역을 엑셀로 정리해야 하며, 세무사에게 보낼 자료를 월말에 몰아서 만든다.
- AI가 자동 저장하지 않고 승인 후보만 만드는 현재 구조는 사업자 지출 정리에도 안전하다.

### 나쁜 것

- 세무/회계 영역으로 깊게 들어가면 법적 책임과 신뢰 리스크가 커진다.
- 사업자 데이터는 개인 가계부보다 민감하다. 사업자번호, 거래처, 세금계산서, 계좌 정보, 매출 정보까지 다루게 될 수 있다.
- 사업 업종마다 카테고리와 증빙 기준이 달라서 처음부터 모든 업종을 만족시키려 하면 범위가 커진다.

### 상관없는 것

- 처음부터 복식부기, 재무제표, 세금신고 파일 생성까지 할 필요는 없다.
- 처음부터 은행/카드사 자동 연동이 없어도 된다. CSV/XLSX 업로드와 영수증 사진만으로도 검증 가능하다.

## 3. 외부 자료 기반으로 확인한 시장/요구사항

### 국내 기준: 간편장부와 증빙 보관

국세청 자료 기준으로 소규모 사업자는 간편장부 대상이 될 수 있으며, 간편장부는 회계지식이 없는 사람도 수입과 비용을 가계부처럼 쉽게 작성할 수 있게 만든 장부다.

확인한 주요 요구사항:

- 사업자는 사업 관련 거래를 장부에 기록하고 증빙서류와 함께 보관해야 한다.
- 국세청 안내 자료에는 관련 증빙서류 보관 기간으로 5년이 언급된다.
- 거래 건당 3만원을 초과하는 사업 관련 지출은 법정 지출증빙서류 수취가 요구된다.
- 법정 지출증빙서류 예시는 세금계산서, 계산서, 신용카드매출전표, 현금영수증 등이다.
- 간편장부 대상 여부는 업종과 직전연도 수입금액 기준에 따라 달라진다.

제품 반영:

- "증빙 있음/없음" 상태를 거래마다 표시한다.
- 월별 "증빙 누락 지출" 목록을 만든다.
- 3만원 초과 지출 중 증빙이 없는 항목을 별도 표시한다.
- 간편장부 양식에 가까운 일자별 수입/비용 내보내기를 제공한다.
- 앱 안에서는 세무 판단을 확정하지 않고 "확인 필요"로 표현한다.

### 해외/일반 지출 관리 앱에서 반복되는 기능

소상공인·프리랜서용 지출 앱들은 대체로 아래 기능을 핵심 가치로 내세운다.

- 영수증 사진 촬영 후 OCR/AI로 가맹점, 날짜, 금액, 세금, 통화 추출
- PDF 청구서/인보이스 업로드
- 이메일 영수증 전달 또는 메일함 연동
- 은행/카드 거래와 영수증 매칭
- 자동 카테고리 분류
- 회계사/세무사 전달용 CSV/XLSX/PDF 리포트
- 원본 영수증 링크 또는 이미지 보관
- 모바일에서 빠른 촬영, 일괄 업로드, 검토 후 저장

제품 반영:

- "영수증 스캔 -> 후보 생성 -> 검토 -> 월별 리포트"를 핵심 흐름으로 잡는다.
- 추후 이메일 영수증 전달 주소, PDF 업로드, 은행거래 매칭을 확장 후보로 둔다.

## 4. 타깃 사용자

### 1차 타깃

- 카페, 음식점, 공방, 미용실, 작은 매장 운영자
- 프리랜서, 1인 사업자
- 배달앱/플랫폼 매출과 카드 지출이 섞이는 소규모 사업자
- 세무사는 쓰지만 자료 정리를 매달 힘들어하는 사업자

### 2차 타깃

- 부부가 같이 운영하는 가게
- 공동사업자
- 매장 여러 개를 아직 회계 시스템 없이 관리하는 소규모 팀

### 처음에는 피할 타깃

- 복식부기와 재무제표가 꼭 필요한 법인
- 직원 지출 승인 워크플로우가 복잡한 회사
- 세금계산서 발행/수취 자동화까지 필요한 사업자
- 업종별 세무 판단이 중요한 고위험 업종

## 5. MVP 범위

### MVP 이름

사업장 지출 정리 모드

### 핵심 흐름

1. 사용자가 가입 후 "사업장 지출 정리" 모드를 선택한다.
2. 사업장명과 업종을 입력한다.
3. 기본 비용 카테고리가 생성된다.
4. 영수증 사진, 카드/계좌 CSV 또는 XLSX를 업로드한다.
5. AI가 지출 후보를 만든다.
6. 사용자가 후보를 검토하고 승인한다.
7. 월별 비용 리포트와 증빙 누락 목록을 본다.
8. 세무사 전달용 XLSX/CSV/PDF를 내보낸다.

### MVP에 포함할 기능

- 사업장 프로필
- 사업장용 기본 카테고리
- 지출/매출 구분
- 영수증 업로드 및 OCR/AI 후보 생성
- 카드/계좌 CSV/XLSX 가져오기
- 후보 승인/거절/수정
- 증빙 있음/없음 상태
- 3만원 초과 증빙 확인 필요 표시
- 월별 지출 합계
- 카테고리별 지출 합계
- 결제수단별 지출 합계
- 증빙 누락 목록
- 세무사 전달용 XLSX/CSV 내보내기

### MVP에 넣지 않을 기능

- 부가세 자동 신고
- 종합소득세 자동 계산
- 경비 인정 여부 확정
- 세금계산서 홈택스 자동 수집
- 은행/카드사 실시간 자동 연동
- 복식부기 자동 생성
- 회계사/세무사 권한 포털

## 9. 개인 가계부와 사업장 장부를 같이 쓰는 구조

소상공인은 개인 생활비와 사업장 비용을 동시에 관리하는 경우가 많다. 이때 가장 중요한 원칙은 "계정은 하나, 데이터 공간은 분리"다.

권장 구조:

> 한 사용자 계정 안에 여러 공간을 만들고, 각 공간을 personal, family, business 타입으로 나눈다.

예시:

- 내 개인 가계부
- 우리 가족 가계부
- 카페 운영비
- 프리랜서 사업장
- 공방 지출 정리

### 좋은 방식

앱 상단 헤더의 기존 개인 가계부/모임 전환 방식을 그대로 확장한다. 새 전환 UI를 따로 만들지 않고, 현재 전환 컴포넌트에 사업장 타입을 추가한다.

예:

- 개인 가계부
- 카페 운영비
- 가족 생활비

표시 방식:

- 개인 가계부
- 모임: 우리 가족
- 사업장: 카페 운영비
- 사업장: 프리랜서 장부

공간을 바꾸면 대시보드, 거래, 예산, 통계, 카테고리, 결제수단, 업로드, 내보내기가 모두 해당 공간 기준으로 바뀐다.

개인 공간에서 중요한 것:

- 생활비
- 식비
- 교통비
- 월급
- 용돈
- 개인 카드
- 가족 공유

사업장 공간에서 중요한 것:

- 매출
- 재료비
- 임대료
- 광고비
- 사업 카드
- 사업 계좌
- 증빙 누락 확인
- 세무사 전달용 내보내기

### 나쁜 방식

개인 거래와 사업 거래를 한 화면에 섞고 카테고리만 개인/사업으로 나누는 방식은 피한다.

이 방식의 문제:

- 개인 예산과 사업 비용 통계가 섞인다.
- 가족 공유와 사업장 공유 권한이 꼬인다.
- 세무사 전달용 내보내기에 개인 거래가 섞일 위험이 있다.
- 사업장 증빙 관리와 개인 소비 기록의 보관 정책이 달라진다.
- 나중에 사업장을 여러 개 만들기 어렵다.

### 같이 보고 싶은 경우

저장은 분리하고, 조회만 합쳐서 본다.

별도 "전체 현황" 화면 후보:

- 개인 현금흐름
- 사업장 현금흐름
- 이번 달 총 지출
- 사업장에서 개인으로 이체한 금액
- 개인 카드로 결제한 사업 지출
- 사업 카드로 결제한 개인 지출 의심 항목
- 공간별 잔액/지출 요약

핵심 원칙:

> 저장은 분리, 조회는 필요할 때만 통합.

### 꼭 필요한 기능

- 거래 이동: 개인 가계부에서 사업장 장부로 이동
- 거래 복사: 개인 카드 결제지만 사업 비용으로도 기록해야 할 때 복사
- 개인카드 사업지출 표시
- 사업카드 개인지출 의심 표시
- 공간별 기본 결제수단
- 공간별 기본 카테고리
- 공간별 예산/리포트/내보내기
- 공간별 권한 관리
- 공간별 AI 학습 규칙 분리

### 가입/온보딩 UX

가입 시 아래 선택지를 제공한다.

1. 개인 가계부만 쓸게요
2. 가족/커플 가계부로 쓸게요
3. 사업장 지출 정리도 같이 쓸게요

3번을 선택하면 기본으로 두 공간을 만든다.

- 개인 가계부
- 사업장 장부

이후 사용자는 기존 헤더 상단바의 개인 가계부/모임 전환 메뉴에서 사업장 장부도 같은 방식으로 선택한다.

전환 메뉴 예시:

- 개인 가계부
- 모임: 우리 가족
- 사업장: 카페 운영비

즉, 사업자 기능은 별도 앱이나 별도 진입점이 아니라 기존 전환 방식에 추가되는 세 번째 공간 타입이다.

## 10. 데이터 설계 방향

### 최소 변경안

기존 household를 사업장 컨텍스트로도 사용한다.

장점:

- 현재 공유/RLS 구조를 재사용할 수 있다.
- 가족 가계부와 사업장 모드를 같은 구조로 관리할 수 있다.
- 공동사업자나 직원 초대 기능으로 확장하기 쉽다.

추가 필드 후보:

- households.type: personal, family, business
- households.owner_user_id
- households.business_name
- households.business_category
- households.business_number_masked
- transactions.context_type: personal, family, business
- transactions.transfer_pair_id
- transactions.origin_transaction_id
- transactions.mixed_use_flag: none, personal_card_business_expense, business_card_personal_expense, needs_review
- transactions.evidence_status: none, attached, required, not_required, needs_review
- transactions.evidence_file_id
- transactions.business_note
- uploaded_files.document_type: receipt, invoice, bank_statement, card_statement, contract, other
- user_learning_rules.household_id 또는 workspace_id
- merchant_learning_rules.household_id 또는 workspace_id

주의:

- 사업자번호 원문 저장은 피하고, 필요하면 마스킹 또는 암호화 정책을 먼저 정해야 한다.
- 세무 관련 판단 필드는 확정값이 아니라 "확인 필요" 상태로 둔다.
- 개인 공간과 사업장 공간의 AI 학습 규칙은 섞지 않는다.
- 세무사 전달용 내보내기는 business 타입 공간의 데이터만 대상으로 한다.

## 11. 보안/개인정보/법적 주의

### 반드시 지킬 것

- AI는 후보만 만들고 자동 최종 저장하지 않는다.
- 사용자가 승인한 항목만 장부에 반영한다.
- 사업자번호, 계좌번호, 카드번호, 전화번호는 원문 저장을 피한다.
- OCR 원문은 기존 정책처럼 짧은 보관 기간을 유지한다.
- 원본 증빙 파일 보관 기간과 삭제 정책을 사용자가 이해할 수 있게 보여준다.
- 세무 판단은 "참고/확인 필요"로만 표현한다.
- 개인 거래와 사업장 거래를 권한/RLS/내보내기에서 분리한다.
- 사업장 멤버나 세무사에게 개인 가계부 데이터가 노출되지 않게 한다.

### 금지 문구

- 세금 신고 자동 완료
- 절세 보장
- 경비 인정 보장
- 세무서 제출 가능 확정 자료
- 회계사 없이 완벽 처리

### 권장 문구

- 지출 정리 보조
- 세무사 전달용 자료 정리
- 증빙 누락 확인
- 월별 비용표 생성
- AI 후보를 사용자가 검토 후 승인

작업:

- 사업장 생성 UI

- 사업장 모드에서 기본 카테고리가 정상 생성되는지 확인
- 공간 전환 시 거래/예산/통계/결제수단이 섞이지 않는지 확인

### 1-1단계: 개인/사업 거래 분리 보조

목표:

- 개인 카드와 사업 카드가 섞이는 현실을 처리한다.

작업:

- 거래 이동 기능
- 거래 복사 기능

검증:

- 이동/복사 후 원본과 대상 공간의 권한이 올바른지 확인
- 세무사 전달용 내보내기에 개인 공간 거래가 섞이지 않는지 확인
- AI 학습 규칙이 공간별로 분리되는지 확인

### 2단계: 증빙 관리

목표:

- 지출과 원본 영수증/파일을 연결하고, 증빙 누락을 쉽게 확인한다.

작업:

검증:

- 삭제 시 연결 상태 정리
- 마스킹 정책 확인

### 3단계: 월말 리포트

목표:

- 사장님이 월말에 "얼마 썼고, 무엇이 빠졌는지" 바로 볼 수 있게 한다.

작업:

- 월별 비용 요약
- 카테고리별/결제수단별 합계
- 증빙 누락 목록
- 지난달 대비 변화
- AI 요약 문장

검증:

- 월 경계 KST 처리
- 매출/지출/이체 구분
- 중복 지출 제외

### 4단계: 세무사 전달용 내보내기

목표:

- 월별 정리 자료를 XLSX/CSV로 내보낸다.

작업:

- 간편장부 참고 양식 내보내기
- 지출 목록 시트
- 증빙 누락 시트
- 카테고리 합계 시트
- 원본 파일 링크 또는 파일명 시트

검증:

- 엑셀 한글 깨짐 없음
- 날짜/금액 형식 유지
- 개인정보 마스킹

### 5단계: 베타 테스트

목표:

- 실제 소상공인 3~5명에게 한 달 자료로 검증한다.

질문:

- 월말 정리 시간이 줄었는가?
- AI 분류가 몇 퍼센트 정도 맞았는가?
- 증빙 누락 목록이 유용했는가?
- 세무사에게 보낼 만한 자료인가?
- 얼마까지 낼 의향이 있는가?

성공 기준:

- 사용자 3명 이상이 실제 한 달 자료를 넣는다.
- 한 명 이상이 다음 달에도 계속 쓰겠다고 말한다.
- 세무사 전달용 내보내기가 실제로 사용된다.

## 16. 결론

소상공인 확장은 가능성이 좋다. 다만 세무 자동화가 아니라 "증빙과 지출을 정리해서 월말 자료를 쉽게 만드는 도구"로 시작해야 한다.

가장 강한 초기 가치는 세 가지다.

1. 영수증과 카드/계좌내역을 한 곳에 모으기
2. AI가 후보를 만들고 사용자가 승인하기
3. 월말에 세무사에게 보낼 자료를 바로 내보내기

이 세 가지가 실제 소상공인 3~5명에게 통하면, 그 다음에 은행연동, 이메일 영수증 수집, 세무사 공유, 업종별 리포트로 확장한다.

개인 가계부를 같이 쓰는 소상공인에게는 한 계정 안에서 개인 공간과 사업장 공간을 분리하는 것이 가장 안전하다. 데이터는 섞지 않고, 필요할 때만 전체 현황에서 통합 조회한다.
