# Execute Plans

비자명한 작업의 **plan-first** 흐름이 모이는 폴더.
AGENTS.md §5 와 CONTRACT.md §9-A-4 (기능 경계 명확히) 의 운영 형태.

## 언제 새 plan 을 작성하나

- 영역 경계를 넘는 변경 (한 에이전트의 Scope를 넘어가는 경우)
- 마이그레이션 / RLS / 시크릿 / 외부 의존성 추가
- AI 프롬프트 / 모델 / OCR 엔진 교체
- 데이터 모델 변경
- UI/UX 의 큰 흐름 재설계 (단순 컴포넌트 추가 아님)

자명한 변경(버그 1줄 픽스, 오타, 한 컴포넌트 내부 정돈)은 plan 없이 바로 PR.

## 작성 규칙

- 파일명: `YYYY-MM-DD-<짧은-제목>.md` (예: `2026-05-08-mobile-push-phase1.md`)
- 절대 삭제 금지 — 과거 plan 은 회고/온보딩 자료
- 진행 중에 의도가 바뀌면 같은 파일에 **갱신 로그** 추가
- 작업 종료 후 `## 결과` / `## 이슈` / `## 다음 액션` 섹션을 채운다
