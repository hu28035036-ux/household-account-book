# UI_UX_STRUCTURE

## 화면 목록
1. 로그인 / 회원가입
2. 대시보드 (`/dashboard`)
3. 거래내역 (`/transactions`)
4. AI 업로드 (`/upload`)
5. 분석 결과 후보 (`/candidates`)
6. 카테고리 관리 (`/categories`)
7. 결제수단 관리 (`/payment-methods`)
8. 설정 (`/settings`)
9. 원본 파일 관리 (`/files`)

## 네비게이션
- **PC**: 좌측 사이드바(아이콘+라벨), 상단 헤더(사용자/알림/검색).
- **모바일**: 상단 슬림 헤더(타이틀+뒤로가기), 하단 탭바(대시보드 / 거래 / 업로드 / 후보 / 설정).
- 같은 기능을 모바일/PC에서 모두 사용 가능.

## 화면별 핵심 구조

### 로그인/회원가입
- Supabase Auth UI(자체 폼) + 매직링크.
- 로그인 후 `/dashboard`로 리다이렉트.

### 대시보드
- 카드형 요약 지표(이번 달 지출/수입/잔액/예산 사용률/AI 후보 대기 수).
- 카테고리별 지출 차트, 결제수단별 지출 차트, 최근 거래 5~10건.
- PC 3~4열 그리드, 모바일 1~2열.

### 거래내역
- 상단 필터바: 기간/카테고리/결제수단/검색어.
- PC: 표(날짜/가맹점/카테고리/결제수단/금액/메모/액션).
- 모바일: 카드 리스트(날짜+가맹점+금액 강조, 카테고리/결제수단은 보조).
- 액션: 수정 모달(공통), 삭제(확인 다이얼로그).

### AI 업로드
- 드래그앤드롭 영역 + "사진 찍기/파일 선택" 버튼(`accept="image/*" capture="environment"`).
- 다중 업로드 가능. 업로드 진행률 표시.
- 업로드 완료 후 OCR 진행 상태 표시.
- OCR 텍스트 미리보기(접기/펼치기) → "AI 분석 시작" 버튼.
- 분석 진행 후 자동으로 후보 화면으로 이동(또는 같은 페이지 하단에 후보 표시).

### 분석 결과 후보
- PC: 표(날짜/가맹점/금액/카테고리 추천/결제수단 추천/신뢰도/중복/근거/액션) + 일괄 승인/제외 버튼 상단.
- 모바일: 카드 리스트.
  - 상단: 가맹점 + 금액(강조)
  - 중간: 날짜 / 카테고리 추천 / 결제수단 추천 / 신뢰도
  - 하단: 수정 / 제외 / 승인
  - 중복·확인 필요 배지 명확히 표시
- 일괄 승인 버튼은 모바일에서 sticky bottom bar(의심 항목은 일괄 대상에서 자동 제외).

### 카테고리 관리
- PC: 좌측 목록 + 우측 수정 폼.
- 모바일: 목록 + 모달/바텀시트 편집.
- 색상/아이콘 선택 UI는 터치 친화적 크기.

### 결제수단 관리
- 카드 리스트(이름/타입/마스킹된 번호/발급사).
- 추가/수정 폼은 모달 또는 바텀시트.
- **카드/계좌번호 입력란은 마지막 4자리만 받는 구조**(전체 입력 막기).

### 설정
- 섹션: 계정 / 보안 / 원본 파일 정책 / AI 서버 설정 / 데이터 내보내기·삭제.
- 위험 버튼(원본 파일 삭제, 계정 삭제)은 확인 다이얼로그 + 텍스트 입력 확인.

### 원본 파일 관리
- 업로드된 파일 리스트(썸네일/이름/상태/날짜).
- 파일 삭제 → Storage + DB 일관 처리.
- 파일별 상태 배지(uploaded / ocr_done / parsed / approved / failed).

## 사용자 액션 모달/다이얼로그 표준
- 일반 편집: 모달(데스크톱) / 바텀시트(모바일).
- 위험 작업: 확인 다이얼로그 + (필요 시) 텍스트 입력 확인.
- 일괄 작업: 토스트로 결과 안내.

## 접근성
- 색상만으로 상태 구분 금지(텍스트/배지 동반).
- 아이콘 단독 버튼은 `aria-label` 필수.
- 키보드 포커스 표시(특히 PC).
- 폰트 크기 본문 ≥ 14px(모바일은 ≥ 16px 권장).

## 빈 상태(Empty State)
- 거래내역 0건 → "첫 영수증을 올려보세요" + 업로드 버튼.
- 후보 0건 → "분석 대기 중인 후보가 없어요" + 업로드 안내.
- 파일 0개 → "업로드한 파일이 없어요".
